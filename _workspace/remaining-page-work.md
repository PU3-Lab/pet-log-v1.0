# 남은 페이지 작업

## 현재 상태

- `app/web`에 Next.js App Router 앱이 구성되어 있습니다.
- 더미 데이터를 사용하는 정적 UI가 다음 경로에 구현되어 있습니다.
  - `/`
  - `/record`
  - `/analysis`
  - `/community`
  - `/more`
  - `/timeline`
  - `/suggestions`
  - `/profile`
- 하단 탭은 참고 이미지 기준인 `홈 / 기록 / 분석 / 커뮤니티 / 더보기` 구조로 맞춰져 있습니다.
- 아직 API, 데이터베이스, 인증, 영구 저장 기능은 없습니다.

## 진행 우선순위

### 1. 기록 입력 상태 연결

- `/record`에서 입력한 기록이 세션 내 데이터에 추가되도록 클라이언트 상태를 연결합니다.
- 저장 직후 입력 내용이 미리보기와 `/timeline`에 반영되도록 만듭니다.
- 기록 입력값의 최소 검증 규칙을 추가합니다.

### 2. 필터와 선택 UI 동작 구현

- 타임라인, 분석, AI 제안, 커뮤니티의 카테고리 필터를 실제 목록 필터와 연결합니다.
- pill, segmented control, 탭 형태 UI에 활성 상태를 적용합니다.
- 자세히 보기, 상단 버튼, 카드 클릭 같은 이동 흐름을 점검하고 연결합니다.

### 3. 데이터 타입 안정화

- 기록, 반려동물 프로필, 분석 지표, AI 제안, 커뮤니티 항목 타입을 분리해 정의합니다.
- 더미 데이터를 타입 기준에 맞춰 정리합니다.
- 이후 API나 저장소를 붙여도 화면 구조가 크게 흔들리지 않도록 데이터 경계를 고정합니다.

### 4. 저장 방식 결정 및 적용

- 빠른 MVP 검증 기준으로는 `localStorage`를 우선 검토합니다.
- 서버 흐름 검증이 필요하면 Next route handler를 검토합니다.
- 실제 제품화 단계에서는 별도 백엔드와 데이터베이스로 확장합니다.

### 5. 기록 및 프로필 CRUD

- 기록 생성, 조회, 수정, 삭제 흐름을 추가합니다.
- 반려동물 프로필 생성과 수정 흐름을 추가합니다.
- 입력 실패, 삭제 확인, 빈 목록 같은 기본 상태를 처리합니다.

### 6. AI MVP 흐름 추가

- 자유 텍스트 기록을 카테고리와 필드로 구조화하는 흐름을 추가합니다.
- 신뢰도가 낮은 결과는 사용자가 확인하거나 수정할 수 있게 처리합니다.
- 기록 누락 감지, 이상 변화 감지, 행동 제안은 진단처럼 보이지 않도록 문구를 제한합니다.

### 7. 상태 화면과 반응형 보강

- 빈 상태, 로딩 상태, 오류 상태를 주요 페이지에 추가합니다.
- 모바일 320px 폭에서 텍스트 줄바꿈, 버튼 크기, 카드 겹침을 확인합니다.
- 콘텐츠가 길어져도 하단 메뉴바가 유지되는 구조를 계속 검증합니다.

### 8. UI 디테일 정리

- `pet-log-ui.png`와 비교해 간격, 카드 높이, 아이콘, 색상 디테일을 보정합니다.
- 임시 SVG 아이콘을 유지할지, 아이콘 라이브러리로 교체할지 결정합니다.
- 홈, 기록, 분석, 커뮤니티, 더보기의 화면 밀도와 시각적 일관성을 맞춥니다.

### 9. 제품 확장 기능 구체화

- 일정, 알림, 설정, 공동 관리, 병원 연계, 커뮤니티 상세 흐름을 MVP 이후 작업으로 분리합니다.
- 커머스, 지도, IoT, 지출 관리는 사용자가 MVP 범위로 명시하기 전까지 후순위로 둡니다.

## 스프린트 계획

### 스프린트 1. 기록 입력과 세션 타임라인 연결

목표: 사용자가 `/record`에서 입력한 기록이 앱 안에서 실제 데이터처럼 이어지게 만듭니다.

상태: 완료

- `/record` 입력 상태와 저장 흐름을 구현합니다.
- 기록 입력값의 필수값, 길이, 카테고리 검증을 추가합니다.
- 저장된 기록을 세션 상태에 추가하고 `/timeline`에 반영합니다.
- 저장 직후 최근 기록 미리보기와 이동 흐름을 연결합니다.
- 기록이 없을 때의 빈 상태를 추가합니다.

완료 기준: 새로 입력한 기록이 새로고침 전 세션에서 홈 또는 타임라인에 표시되고, 잘못된 입력은 저장되지 않습니다.

결과: 공용 클라이언트 기록 상태를 추가했고, `/record`에서 저장한 내용이 홈 최근 기록과 `/timeline`에 즉시 반영됩니다.

### 스프린트 2. 필터, 선택 UI, 데이터 타입 안정화

목표: 현재 정적 UI를 실제 조작 가능한 MVP 화면으로 정리합니다.

상태: 완료

- 기록, 반려동물 프로필, 분석 지표, AI 제안, 커뮤니티 항목 타입을 분리합니다.
- 더미 데이터를 타입 기준에 맞춰 정리합니다.
- 타임라인, 분석, AI 제안, 커뮤니티 필터를 실제 목록 필터와 연결합니다.
- pill, segmented control, 탭 UI의 활성 상태를 구현합니다.
- 카드 클릭, 자세히 보기, 상단 버튼의 이동 흐름을 점검합니다.

완료 기준: 주요 목록 화면에서 사용자의 선택에 따라 표시 데이터가 바뀌고, 타입 기준이 이후 저장소 연결에 재사용될 수 있습니다.

결과: 공유 타입을 `app/web/src/lib/types.ts`로 분리했고, 타임라인/분석/제안/커뮤니티의 필터와 탭이 실제 표시 데이터를 바꾸도록 연결했습니다.

### 스프린트 3. 저장 방식과 CRUD MVP

목표: 데모 수준을 넘어 기본 사용 흐름을 유지할 수 있는 저장 구조를 붙입니다.

상태: 완료

- MVP 저장 방식으로 `localStorage` 우선 적용 여부를 확정합니다.
- 기록 생성, 조회, 수정, 삭제 흐름을 구현합니다.
- 반려동물 프로필 생성과 수정 흐름을 구현합니다.
- 삭제 확인, 저장 실패, 빈 목록 상태를 처리합니다.
- 저장 데이터 버전 또는 기본 마이그레이션 방식을 간단히 정의합니다.

완료 기준: 브라우저를 새로고침해도 기본 기록과 프로필이 유지되고, 사용자가 기록과 프로필을 수정하거나 삭제할 수 있습니다.

결과: `localStorage` 기반 저장 스냅샷을 추가했고, 기록은 생성/조회/수정/삭제가 가능하며 프로필은 편집 후 홈과 프로필 화면에 유지됩니다.

### 스프린트 4. AI MVP 흐름과 반응형 QA

목표: Pet Log의 핵심 차별점인 AI 기록 보조를 안전한 표현과 검증 가능한 UI로 추가합니다.

상태: 완료

- 자유 텍스트 기록을 카테고리와 필드로 구조화하는 흐름을 추가합니다.
- 신뢰도가 낮은 결과는 사용자가 확인하거나 수정할 수 있게 처리합니다.
- 기록 누락 감지, 이상 변화 감지, 행동 제안 문구를 추가합니다.
- AI 제안은 진단처럼 보이지 않도록 안전 문구를 제한합니다.
- 320px 모바일, 긴 콘텐츠, 하단 메뉴 고정, 뒤로가기 흐름을 다시 검증합니다.

완료 기준: AI 제안이 저장된 기록을 기반으로 표시되고, 사용자가 오해할 수 있는 진단성 표현 없이 모바일 주요 화면이 깨지지 않습니다.

결과: 규칙 기반 AI MVP 유틸을 추가했고, `/record`는 구조화 미리보기와 신뢰도 확인을 제공하며 `/analysis`와 `/suggestions`는 저장된 기록 기반의 누락/주의/안정 인사이트와 안전 문구를 표시합니다.

### 스프린트 5. MVP 이후 확장 정리

목표: 당장 만들 기능과 제품화 이후 확장 기능을 분리해 범위가 흔들리지 않게 합니다.

- 일정, 알림, 설정, 공동 관리, 병원 연계, 커뮤니티 상세 흐름을 별도 후보로 정리합니다.
- 커머스, 지도, IoT, 지출 관리는 후순위 확장으로 유지합니다.
- API, 백엔드, 데이터베이스 전환 시 필요한 경계와 마이그레이션 항목을 기록합니다.
- 배포 전 점검 항목과 테스트 범위를 정리합니다.

완료 기준: MVP에 포함할 작업과 다음 릴리스 후보가 문서상 명확히 분리됩니다.
