# 남은 페이지 작업

## 현재 상태

- `app/web`에 Next.js App Router 앱이 구성되어 있습니다.
- 더미 데이터를 사용하는 정적 UI가 다음 경로에 구현되어 있습니다.
  - `/`
  - `/record`
  - `/analysis`
  - `/community`
  - `/more`
  - `/timeline`
  - `/suggestions`
  - `/profile`
- 하단 탭은 참고 이미지 기준인 `홈 / 기록 / 분석 / 커뮤니티 / 더보기` 구조로 맞춰져 있습니다.
- 아직 API, 데이터베이스, 인증, 영구 저장 기능은 없습니다.

## 남은 작업

### 1. UI 완성도 보강

- `pet-log-ui.png`와 비교해 간격, 카드 높이, 아이콘 처리, 색상 디테일을 조정합니다.
- 주요 페이지별 빈 상태, 로딩 상태, 오류 상태를 추가합니다.
- 모바일 320px 폭에서 텍스트 줄바꿈과 버튼/카드 겹침을 확인합니다.
- 아이콘 라이브러리를 추가한다면 현재 임시 아이콘 표현을 일관된 아이콘 시스템으로 교체합니다.

### 2. 클라이언트 상호작용

- 기록 입력 내용이 미리보기와 세션 내 타임라인에 반영되도록 만듭니다.
- 타임라인, 분석, 제안, 커뮤니티의 카테고리 필터가 실제로 동작하게 합니다.
- 상단 버튼, 뒤로가기, 자세히 보기 버튼의 이동 또는 상태 변경을 연결합니다.
- pill, segmented control 같은 선택 UI에 활성 상태를 추가합니다.

### 3. 데이터와 API

- 더미 데이터를 기록, 반려동물 프로필, 인사이트, 제안, 지표 타입으로 안정화합니다.
- 첫 저장 방식으로 Next route handler, local storage, 별도 백엔드 중 무엇을 쓸지 결정합니다.
- 반려동물 프로필과 기록의 생성, 조회, 수정 흐름을 추가합니다.
- 사용자가 입력한 기록에 대한 검증과 오류 처리 방식을 정의합니다.

### 4. AI MVP 동작

- 자유 텍스트 기록을 카테고리와 필드로 구조화하는 흐름을 설계합니다.
- 신뢰도가 낮은 추출 결과는 사용자가 확인하거나 수정할 수 있는 UI를 추가합니다.
- 저장된 기록을 기준으로 기록 누락 감지와 이상 변화 감지를 구현합니다.
- AI 제안은 확정 진단처럼 보이지 않게 하고, 지속되거나 심한 증상은 병원 상담을 권장하는 문구로 처리합니다.

### 5. 제품 확장

- 일정, 알림, 설정, 공동 관리, 병원 연계, 커뮤니티 상세 흐름을 구체화합니다.
- 커머스, 지도, IoT, 지출 관리는 사용자가 MVP 범위로 명시하기 전까지 후순위로 둡니다.

## 추천 다음 단계

현재 정적 UI 작업을 먼저 커밋한 뒤, 기록 입력, 필터, 탭/상세 이동 같은 클라이언트 상호작용을 붙이고 그다음 저장 기능을 추가하는 순서가 좋습니다.
