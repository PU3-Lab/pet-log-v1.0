# 모바일 UI QA 결과

## 검토 일시

- 2026-05-01

## 검토 범위

- 기준 URL: `http://localhost:3000`
- 뷰포트: `320x740`, `375x812`, `430x932`
- 대상 화면: 홈, 기록 입력, 분석, 타임라인, 프로필, 병원 연계, 쇼핑, 홈 챗봇 바텀시트
- 산출물: `_workspace/qa-screenshots/`

## 확인 결과

- 주요 화면은 320px, 375px, 430px 폭에서 가로 오버플로가 발생하지 않았습니다.
- 하단 네비게이션은 모든 검토 화면에서 뷰포트 안에 표시되었습니다.
- 홈 챗봇 바텀시트는 320px, 375px, 430px 폭에서 입력창이 화면 안에 유지되었습니다.
- 기록 입력, 분석, 타임라인, 프로필, 병원 연계, 쇼핑 화면은 스크린샷 캡처와 기본 렌더링 검사를 통과했습니다.

## 수정한 문제

- 320px 홈 화면에서 `물어보기` 버튼이 `오늘 알림` 제목과 알림 카드 일부를 덮는 문제가 확인되었습니다.
- 320px 이하에서는 `물어보기` 버튼을 AI 질문 카드 아래 인라인 버튼으로 표시하고, 361px 이상에서는 기존 플로팅 버튼을 유지하도록 수정했습니다.

## 남은 참고 사항

- 375px 이상에서는 `물어보기` 플로팅 버튼이 하단 알림 카드 영역 위에 걸칠 수 있으나, 주요 제목과 액션 링크는 가리지 않습니다.
- 개발 서버의 React DevTools 안내와 favicon 404 로그가 확인되었지만 앱 기능 오류는 아닙니다.
- 이번 QA는 headless Chrome 기반 스크린샷 검토이며, 실제 iOS Safari 주소창과 키보드 동작은 실기기 확인이 별도로 필요합니다.
